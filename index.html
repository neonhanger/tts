Overview

A local GPU-accelerated voice cloning service. Clone any voice from a short audio sample and generate speech in that voice. No cloud APIs, no per-request costs, full privacy.

Requirements

  • Python 3.12 (3.14 is too new for some dependencies)
  • NVIDIA GPU with CUDA support (tested on RTX 5080)
  • FFmpeg shared build (with DLLs, not static) — required by torchcodec
  • ~6GB VRAM when model is loaded
  • ~3GB disk for model weights (downloaded on first run)

Quick Start

# Windows
run.bat

# Linux / manual
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu128
python server.py

The server starts on http://0.0.0.0:8000. Open it in a browser to use the web UI.

Features

  • Multi-engine — Flow (F5-TTS), Cosy (CosyVoice2), Higgs (Higgs Audio V2), Ensemble
  • Voice quality scoring — auto-rates uploaded samples on duration, SNR, clipping
  • Multi-reference mode — upload longer audio, auto-split into reference clips
  • Voice mixing — blend two voice profiles with adjustable ratio
  • Emotion control — tag reference clips with emotions, select per request
  • SSML support — <break> tags for pauses, strips other SSML tags
  • Multi-language — auto-detects input language via Unicode script analysis
  • Streaming — poll-based or WebSocket real-time chunk delivery
  • Post-processing — loudness normalization + high-pass filter via ffmpeg
  • Response caching — LRU cache for identical requests
  • Cherry-pick — generate multiple takes, return the best by quality score
  • WhatsApp integration — webhook for text-to-voice-message (scaffold)
  • Fine-tuning — endpoint scaffold for future model fine-tuning

Voice Profiles

Stored in ./voices/. Each contains:

  • reference.wav — the reference audio
  • meta.json — voice ID, metadata, quality score
  • ref_text.txt — pre-transcribed reference text
  • refs/ — multi-reference clips (optional)
  • emotions.json — emotion tags per reference clip (optional)

SSML Support

Enable the SSML toggle to use basic SSML tags in your text input:

<break time="500ms"/>  — insert a pause
<break time="1s"/>     — longer pause
<emphasis>word</emphasis> — stripped (no engine support yet)

Post-Processing Pipeline

When enabled, each generated audio chunk passes through:

  • High-pass filter (80 Hz) — removes rumble and low-frequency noise
  • Loudness normalization (LUFS -16) — consistent volume across chunks

WhatsApp Integration

Set up a Meta WhatsApp Business account and configure the webhook URL to /whatsapp/webhook. Users can send text and receive voice messages. Use !voice voice_id text to select a voice.

API Reference

GET /health

{ "status": "ok", "engines": {...}, "queue_size": 0, "cache_size": 5 }

GET /engines

[{ "id": "flow", "name": "Flow (fast, reliable)", "loaded": true }]

POST /detect-language

Detect language from text.

curl -X POST http://localhost:8000/detect-language -d "text=Hello world"
{ "language": "en", "confidence": 1.0, "script": "latin" }

GET /voices

[{ "voice_id": "default", "original_filename": "...", "quality": {...} }]

POST /voices

Upload voice sample. Auto-transcribes and quality-scores on upload.

curl -X POST http://localhost:8000/voices -F "file=@voice.wav"

GET /voices/{voice_id}/quality

Get quality analysis for a voice's reference audio.

{ "score": 85, "grade": "excellent", "duration": 7.2, "issues": [], "tips": [] }

GET /voices/{voice_id}/preview

Returns the reference audio WAV.

DELETE /voices/{voice_id}

curl -X DELETE http://localhost:8000/voices/abc123

POST /voices/{voice_id}/refs

Upload longer audio for multi-reference mode. Auto-splits into clips.

GET /voices/{voice_id}/refs

List reference clips with durations and emotion tags.

POST /voices/{voice_id}/refs/{ref_name}/emotion?emotion=happy

Tag a reference clip with an emotion label.

POST /voices/mix

Blend two voice profiles.

curl -X POST http://localhost:8000/voices/mix \
  -H "Content-Type: application/json" \
  -d '{"voice_a": "abc123", "voice_b": "def456", "ratio": 0.6}'

POST /voices/{voice_id}/train

Start fine-tuning (scaffold — returns 501).

POST /speak

Generate complete MP3. Supports emotion, SSML, caching, post-processing.

curl -X POST http://localhost:8000/speak \
  -H "Content-Type: application/json" \
  -d '{"text": "Hello", "voice_id": "default", "emotion": "calm", "ssml": false, "postprocess": true, "cache": true}' \
  --output speech.mp3

Response headers: X-Cache: hit|miss, X-Language: en

POST /speak/candidates?takes=3

Cherry-pick: generate multiple takes, return best by quality score.

Response headers: X-Candidates: [{"take":1,"score":82.3},...], X-Best-Score: 82.3

POST /speak/start

Start streaming job (poll-based).

{ "job_id": "abc123", "total_chunks": 3 }

GET /speak/poll/{job_id}?after=0

Poll for new audio chunks.

WebSocket /ws/speak

Real-time streaming. Send JSON config, receive binary WAV chunks.

// Connect
const ws = new WebSocket('ws://localhost:8000/ws/speak');
ws.onopen = () => ws.send(JSON.stringify({
  text: 'Hello world',
  voice_id: 'default',
  engine: 'flow',
  emotion: 'calm',
  ssml: false,
  postprocess: true
}));
// Receive: JSON messages (info/progress/done/error) + binary WAV chunks

GET /cache/stats

{ "size": 5, "max_size": 100 }

DELETE /cache

Clear all cached responses.

GET /whatsapp/webhook

WhatsApp verification endpoint.

POST /whatsapp/webhook

Incoming WhatsApp message handler. Generates TTS and replies with voice message.

Voice command: !voice pride_prejudice Hello there

Authentication

Header: X-API-Key: your-key. Only required if API_KEY env var is set.
