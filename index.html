Overview

A local GPU-accelerated voice cloning service. Clone any voice from a short audio sample and generate speech in that voice. No cloud APIs, no per-request costs, full privacy.

Requirements

  • Python 3.12 (3.14 is too new for some dependencies)
  • NVIDIA GPU with CUDA support (tested on RTX 5080)
  • FFmpeg shared build (with DLLs, not static) — required by torchcodec
  • ~6GB VRAM when model is loaded
  • ~3GB disk for model weights (downloaded on first run)

Quick Start

# Windows
run.bat

# Linux / manual
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu128
python server.py

The server starts on http://0.0.0.0:8000. Open it in a browser to use the web UI.

Windows Setup Notes

PyTorch CUDA: pip installs CPU-only PyTorch by default. Reinstall with CUDA:

pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu128 --force-reinstall

FFmpeg shared DLLs:

winget install Gyan.FFmpeg.Shared

The run.bat script handles adding the DLL path automatically.

Authentication is off by default for local use. Set API_KEY when exposing to other devices:

# Windows
set API_KEY=my-secret-key
python server.py

# Linux
API_KEY=my-secret-key python server.py

Voice Profiles

Stored in ./voices/. Each contains:

  • reference.wav — the reference audio
  • meta.json — voice ID and metadata
  • ref_text.txt — pre-transcribed reference text (speeds up inference)

For best results: 5-10 seconds, clear speech, minimal background noise. You can preview the reference audio before selecting a voice.

Streaming

When streaming is enabled, audio is generated chunk by chunk and sent to the browser via Server-Sent Events (SSE). You hear audio within seconds of hitting Generate, even for long text. Each sentence chunk is generated independently and played sequentially.

Text Chunking

Long text is automatically split at sentence boundaries (~200 chars each). Each chunk is generated separately for better quality and enables streaming playback.

Retry Logic

Each chunk is retried up to 2 times on failure. If generation still fails, use the Retry button to try again. The GPU lock ensures only one generation runs at a time — additional requests queue up.

API Reference

GET /health

{ "status": "ok", "model_loaded": true, "queue_size": 0 }

GET /voices

[{ "voice_id": "default", "original_filename": "basic_ref_en.wav", "ref_text": "..." }]

POST /voices

Upload voice sample. Reference audio is auto-transcribed on upload for faster inference.

curl -X POST http://localhost:8000/voices -F "file=@voice.wav"

GET /voices/{voice_id}/preview

Returns the reference audio WAV for previewing the voice.

DELETE /voices/{voice_id}

curl -X DELETE http://localhost:8000/voices/abc123

POST /speak

Generate complete MP3. Blocks until all chunks are done.

curl -X POST http://localhost:8000/speak \
  -H "Content-Type: application/json" \
  -d '{"text": "Hello world", "voice_id": "default"}' \
  --output speech.mp3

POST /speak/stream

Stream generation via Server-Sent Events. Each chunk is sent as a base64-encoded MP3.

Events:

EventData
info{ "job_id": "...", "total_chunks": 3 }
progress{ "chunk": 1, "total": 3, "text": "..." }
audio{ "chunk": 1, "total": 3, "audio_b64": "..." }
done{ "status": "complete" }
error{ "detail": "..." }
// JavaScript example
const es = new EventSource('/speak/stream', { method: 'POST', ... });
// Note: EventSource is GET-only. Use fetch with ReadableStream instead:
const resp = await fetch('/speak/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ text: '...', voice_id: 'default' })
});
// Parse SSE from response body stream
